# TP 2 — Endpoint Express de traitement de gros fichier en streaming

> **Module 2** — Streams Node.js
> **Durée** : environ 1h30
> **Difficulté** : 4/5
> **Type** : TP guidé

## Mise en situation

Votre équipe expose une API interne de traitement de logs. Les clients envoient des fichiers de plusieurs centaines de mégaoctets (parfois plusieurs gigaoctets) à un endpoint qui doit :

1. recevoir le fichier en **streaming** (sans le bufferiser intégralement),
2. le transformer à la volée (anonymisation et mise en forme),
3. en écrire une copie sur disque **et** renvoyer le résultat transformé dans la réponse HTTP.

Toute la chaîne doit être assemblée avec `stream.pipeline` et résister aux erreurs (client qui coupe la connexion, fichier corrompu, disque plein).

## Objectifs

- Recevoir un corps de requête HTTP comme un Readable stream dans Express
- Insérer un Transform stream personnalisé dans la chaîne
- Diffuser le résultat à la fois vers un `WriteStream` fichier et vers la réponse HTTP
- Orchestrer le tout avec `stream.pipeline` et gérer toutes les erreurs sans fuite de ressources

## Prérequis techniques

- Node.js 20 ou 22 et npm
- Express 4 ou 5
- Connaissance des sections 1 à 3 du module

Vérifiez votre environnement :

```bash
node --version            # doit afficher v20.x ou v22.x
npm --version
mkdir tp-streams && cd tp-streams

```

## Architecture cible

```
                +-----------------------------------------------+
                |                Express server                 |
  POST /process |                                               |
  (req body) -->|  req (Readable)                               |
                |     |                                         |
                |     v                                         |
                |  Anonymizer (Transform)                       |
                |     |                                         |
                |     +-----------------+                       |
                |     |                 |                       |
                |     v                 v                       |
                |  res (Writable)   fichier de sortie           |
                |  (réponse HTTP)   (fs.WriteStream)            |
                +-----------------------------------------------+
```

Note : pour diffuser vers deux destinations simultanément, on insère un `PassThrough` ou on duplique le flux après transformation.

## Étapes

### Étape 1 — Serveur Express et endpoint réceptionnant un stream (durée : 25 min)

**Objectif :** disposer d'un endpoint `POST /process` qui lit le corps de la requête comme un stream et le renvoie tel quel, sans transformation, via `pipeline`.

Créez `server.js` :

```js
const express = require('express');
const { pipeline } = require('node:stream/promises');

const app = express();

app.post('/process', async (req, res) => {
  res.setHeader('Content-Type', 'text/plain; charset=utf-8');
  try {
    await pipeline(req, res);
  } catch (err) {
    console.error('Erreur pipeline :', err.message);
    if (!res.headersSent) {
      res.status(500).end('Erreur de traitement');
    } else {
      res.destroy(err);
    }
  }
});

app.listen(3000, () => console.log('En écoute sur http://localhost:3000'));
```

Démarrez et testez avec un fichier :

```bash
node server.js
# dans un autre terminal :
printf 'id,name\n1,alice\n2,bob\n' > sample.txt
curl --data-binary @sample.txt http://localhost:3000/process
```

**Point de contrôle :** la réponse `curl` affiche exactement le contenu envoyé. Ne pas utiliser `express.json()` ni `express.text()` sur cette route, sinon le corps serait bufferisé avant d'arriver au stream.

### Étape 2 — Insérer un Transform d'anonymisation (durée : 30 min)

**Objectif :** ajouter dans la chaîne un Transform qui remplace toute adresse e-mail par `***@***` et passe les noms en majuscules, ligne par ligne.

Créez `anonymizer.js` :

```js
const { Transform } = require('node:stream');

const EMAIL = /[a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,}/gi;

class Anonymizer extends Transform {
  constructor(options) {
    super(options);
    this.tail = '';
  }

  _transform(chunk, encoding, callback) {
    const data = this.tail + chunk.toString('utf8');
    const lines = data.split('\n');
    this.tail = lines.pop(); // dernière ligne potentiellement incomplète
    for (const line of lines) {
      this.push(this.clean(line) + '\n');
    }
    callback();
  }

  _flush(callback) {
    if (this.tail.length > 0) {
      this.push(this.clean(this.tail));
    }
    callback();
  }

  clean(line) {
    return line.replace(EMAIL, '***@***').toUpperCase();
  }
}

module.exports = { Anonymizer };
```

Modifiez l'appel `pipeline` dans `server.js` :

```js
const { Anonymizer } = require('./anonymizer');
// ...
await pipeline(req, new Anonymizer(), res);
```

Testez :

```bash
node server.js
printf 'jean dupont <jean@acme.fr>\nmarie curie <marie@lab.org>\n' > people.txt
curl --data-binary @people.txt http://localhost:3000/process
```

**Point de contrôle :** la réponse affiche les lignes en majuscules avec les e-mails remplacés par `***@***`. Aucune ligne n'est tronquée même si vous envoyez un gros fichier.

### Étape 3 — Diffuser vers le fichier ET la réponse, avec gestion d'erreurs (durée : 35 min)

**Objectif :** écrire le résultat transformé dans un fichier de sortie tout en le renvoyant dans la réponse HTTP, et couvrir tous les cas d'erreur.

Insérez un `PassThrough` après le Transform pour dupliquer le flux vers deux destinations, et orchestrez avec deux `pipeline` parallèles :

```js
const fs = require('node:fs');
const { PassThrough } = require('node:stream');
const { pipeline } = require('node:stream/promises');
const { Anonymizer } = require('./anonymizer');

app.post('/process', async (req, res) => {
  res.setHeader('Content-Type', 'text/plain; charset=utf-8');

  const outPath = `./output-${Date.now()}.txt`;
  const fileStream = fs.createWriteStream(outPath);
  const tee = new PassThrough();

  try {
    await Promise.all([
      pipeline(req, new Anonymizer(), tee),
      pipeline(tee, res),
      pipeline(tee, fileStream),
    ]);
    console.log('Fichier écrit :', outPath);
  } catch (err) {
    console.error('Erreur de traitement :', err.message);
    fileStream.destroy();
    fs.promises.unlink(outPath).catch(() => {});
    if (!res.headersSent) {
      res.status(500).end('Erreur de traitement');
    } else {
      res.destroy(err);
    }
  }
});
```

Attention : un `PassThrough` ne peut alimenter qu'un seul consommateur via `pipeline`. Pour alimenter deux destinations, branchez `tee.pipe(res)` et `tee.pipe(fileStream)` AVANT de lancer `pipeline(req, new Anonymizer(), tee)`, ou créez deux `PassThrough` distincts écrits dans le `_transform`. Choisissez l'approche, justifiez-la et corrigez le squelette en conséquence.

Testez le cas nominal puis le cas d'interruption :

```bash
node server.js
curl --data-binary @people.txt http://localhost:3000/process   # nominal
# interruption : couper curl pendant l'envoi d'un gros fichier (Ctrl+C)
```

**Point de contrôle :** en cas nominal, le fichier `output-*.txt` contient le même contenu que la réponse. En cas d'interruption client, le serveur ne plante pas, log l'erreur, et le fichier partiel est supprimé. Aucun descripteur de fichier ne reste ouvert.

## Dépannage

<details>
<summary>La réponse est vide ou tronquée</summary>

Vérifiez que vous n'avez pas de middleware `express.json()` / `express.text()` global qui consomme le corps de la requête avant votre route. Placez ces middlewares uniquement sur les routes qui en ont besoin.

</details>

<details>
<summary>Des lignes sont coupées en plein milieu</summary>

C'est la gestion du `this.tail` qui est en cause : un chunk peut couper une ligne. Assurez-vous de conserver la dernière ligne incomplète dans `this.tail` et de la traiter dans `_flush`.

</details>

<details>
<summary>« write after end » ou flux qui ne se terminent pas</summary>

Un `PassThrough` branché sur plusieurs consommateurs propage `end` à tous. Si vous utilisez `pipeline(tee, res)` et `pipeline(tee, fileStream)` en parallèle, sachez que `pipeline` consomme la source : deux pipelines lisant le même `tee` se disputeront les chunks. Préférez `tee.pipe(res)` + `tee.pipe(fileStream)`, ou deux PassThrough alimentés depuis le `_transform`.

</details>

<details>
<summary>Le fichier partiel n'est pas supprimé en cas d'erreur</summary>

Appelez `fileStream.destroy()` avant `fs.promises.unlink`, et ignorez l'erreur d'`unlink` (le fichier peut ne pas exister encore). Faites-le dans le bloc `catch`.

</details>

## Pour aller plus loin

- Ajoutez une limite de taille (rejet 413 au-delà d'un seuil) en comptant les octets dans un Transform intermédiaire.
- Compressez la réponse à la volée en insérant `zlib.createGzip()` et le header `Content-Encoding: gzip`.
- Exposez une métrique du débit (octets/seconde) calculée pendant le traitement.
- Mesurez la consommation mémoire avec `process.memoryUsage().rss` pendant l'envoi d'un fichier de 1 Go et vérifiez qu'elle reste stable.
