# TP 3 — Couvrir une API Express de tests, du unitaire au fonctionnel

> Module 3 — Tests avec Node.js
> Durée : environ 1h30
> Difficulté : 4/5
> Type : TP guidé

## Mise en situation

Vous reprenez la maintenance d'une petite API Express de gestion de tâches
(`/tasks`). Elle n'a aucun test. Votre mission : la rendre testable et la couvrir
à plusieurs niveaux — tests unitaires d'un service, tests asynchrones de routes
HTTP avec une dépendance externe stubbée, et un test fonctionnel headless d'un
parcours utilisateur. Vous mettrez en place la suite avec **Mocha** ou **Jest**,
les deux configurations étant fournies.

## Objectifs

- Structurer un projet Express testable (injection de dépendances)
- Tester des routes asynchrones avec supertest
- Isoler une API externe avec un stub sinon
- Écrire un test fonctionnel headless avec Playwright
- Configurer la suite de tests au choix avec Mocha ou Jest

## Prérequis techniques

- Node.js 20 ou 22, npm 10+
- Connaissance d'Express et de `async/await`
- Bases vues aux sections 1 à 3 du module

```bash
mkdir tp-tests-api && cd tp-tests-api
npm init -y
npm install express
npm install -D supertest sinon
# Choisir UN framework :
npm install -D mocha          # option Mocha (+ assert natif)
# OU
npm install -D jest           # option Jest
# Pour le test fonctionnel :
npm install -D @playwright/test
npx playwright install chromium
```

## Étape 1 — Rendre l'API testable

Durée : environ 25 minutes
Objectif : séparer l'app du serveur et injecter les dépendances.

Créez `src/weather-api.js` (la dépendance externe) :

```js
// src/weather-api.js
export class WeatherApi {
  async forecast(city) {
    // En production : appel HTTP réel. Ici, simplifié.
    return { city, temp: 20 };
  }
}
```

Créez `src/task-service.js` avec injection :

```js
// src/task-service.js
export class TaskService {
  constructor(weatherApi) {
    this.weatherApi = weatherApi;
    this.tasks = [];
    this.nextId = 1;
  }

  async create({ title, city }) {
    if (!title) throw new Error("TITLE_REQUIRED");
    const weather = await this.weatherApi.forecast(city);
    const task = { id: this.nextId++, title, city, temp: weather.temp };
    this.tasks.push(task);
    return task;
  }

  list() {
    return this.tasks;
  }
}
```

Créez `src/app.js` (exporte l'app, ne fait pas `listen`) :

```js
// src/app.js
import express from "express";
import { TaskService } from "./task-service.js";
import { WeatherApi } from "./weather-api.js";

export function createApp(taskService = new TaskService(new WeatherApi())) {
  const app = express();
  app.use(express.json());
  app.use(express.static("public"));

  app.get("/tasks", (req, res) => {
    res.json(taskService.list());
  });

  app.post("/tasks", async (req, res) => {
    try {
      const task = await taskService.create(req.body);
      res.status(201).json(task);
    } catch (err) {
      res.status(400).json({ error: err.message });
    }
  });

  return app;
}
```

Créez `src/server.js` :

```js
// src/server.js
import { createApp } from "./app.js";
createApp().listen(3000, () => console.log("API sur http://localhost:3000"));
```

Ajoutez `"type": "module"` dans `package.json`.

Point de contrôle : `node src/server.js` démarre l'API et `curl localhost:3000/tasks`
renvoie `[]`.

## Étape 2 — Tests unitaires, asynchrones et stubs

Durée : environ 35 minutes
Objectif : tester le service en isolant l'API externe, puis les routes via supertest.

Test unitaire du service avec un stub sinon (`tests/task-service.test.js`) :

```js
import { describe, it, afterEach } from "node:test"; // ou depuis mocha/jest
import assert from "node:assert/strict";
import sinon from "sinon";
import { TaskService } from "../src/task-service.js";

describe("TaskService", () => {
  afterEach(() => sinon.restore());

  it("crée une tâche enrichie de la météo (async, stub)", async () => {
    const weatherApi = { forecast: sinon.stub().resolves({ temp: 12 }) };
    const service = new TaskService(weatherApi);

    const task = await service.create({ title: "Acheter", city: "Lyon" });

    assert.equal(task.temp, 12);
    assert.ok(weatherApi.forecast.calledOnceWith("Lyon"));
  });

  it("rejette une tâche sans titre", async () => {
    const weatherApi = { forecast: sinon.stub() };
    const service = new TaskService(weatherApi);
    await assert.rejects(() => service.create({}), /TITLE_REQUIRED/);
    assert.equal(weatherApi.forecast.callCount, 0);
  });
});
```

Test des routes avec supertest (`tests/tasks.routes.test.js`) :

```js
import { describe, it } from "node:test";
import assert from "node:assert/strict";
import sinon from "sinon";
import request from "supertest";
import { createApp } from "../src/app.js";
import { TaskService } from "../src/task-service.js";

describe("Routes /tasks", () => {
  it("POST /tasks crée une tâche (200) avec API externe stubbée", async () => {
    const weatherApi = { forecast: sinon.stub().resolves({ temp: 30 }) };
    const app = createApp(new TaskService(weatherApi));

    const res = await request(app)
      .post("/tasks")
      .send({ title: "Réviser", city: "Nice" });

    assert.equal(res.status, 201);
    assert.equal(res.body.temp, 30);
  });

  it("POST /tasks sans titre renvoie 400", async () => {
    const app = createApp(new TaskService({ forecast: sinon.stub() }));
    const res = await request(app).post("/tasks").send({});
    assert.equal(res.status, 400);
  });
});
```

Configuration Mocha (`.mocharc.json`) :

```json
{ "spec": "tests/**/*.test.js", "timeout": 5000 }
```

Lancement : `npx mocha`. (Avec node:test : `node --test`.)

Configuration Jest (`jest.config.js`) — adapter les imports vers `@jest/globals` :

```js
export default {
  testMatch: ["**/tests/**/*.test.js"],
  testTimeout: 5000,
  transform: {}
};
```

Lancement : `npx jest`.

Point de contrôle : les 4 tests (service + routes) passent, sans accès réseau réel.

## Étape 3 — Test fonctionnel headless

Durée : environ 30 minutes
Objectif : valider un parcours utilisateur dans un navigateur headless.

Créez `public/index.html` (page minimale lisant l'API) :

```html
<!doctype html>
<html lang="fr">
  <body>
    <h1 id="title">Mes tâches</h1>
    <input id="task-title" placeholder="Titre" />
    <button id="add">Ajouter</button>
    <ul id="list"></ul>
    <script>
      document.getElementById("add").addEventListener("click", async () => {
        const title = document.getElementById("task-title").value;
        const res = await fetch("/tasks", {
          method: "POST",
          headers: { "Content-Type": "application/json" },
          body: JSON.stringify({ title, city: "Paris" })
        });
        const task = await res.json();
        const li = document.createElement("li");
        li.textContent = task.title;
        document.getElementById("list").appendChild(li);
      });
    </script>
  </body>
</html>
```

Test Playwright (`tests/tasks.e2e.test.js`) :

```js
import { test, expect } from "@playwright/test";

test("ajoute une tâche depuis l'interface", async ({ page }) => {
  await page.goto("http://localhost:3000/");
  await page.fill("#task-title", "Préparer la démo");
  await page.click("#add");
  await expect(page.locator("#list li")).toHaveText("Préparer la démo");
});
```

Configuration Playwright (`playwright.config.js`) avec démarrage auto du serveur :

```js
import { defineConfig } from "@playwright/test";
export default defineConfig({
  testDir: "tests",
  testMatch: "**/*.e2e.test.js",
  use: { headless: true },
  webServer: {
    command: "node src/server.js",
    url: "http://localhost:3000",
    reuseExistingServer: true
  }
});
```

Lancement : `npx playwright test`.

Point de contrôle : le navigateur headless ajoute la tâche et l'assertion sur la
liste passe ; aucun navigateur ne s'ouvre à l'écran.

## Dépannage

<details>
<summary>"Cannot use import statement outside a module"</summary>

Ajoutez `"type": "module"` dans `package.json`. Pour Jest, l'ESM nécessite de
lancer avec `NODE_OPTIONS=--experimental-vm-modules npx jest` ou d'utiliser la
configuration `transform: {}` montrée plus haut.
</details>

<details>
<summary>Le test Playwright échoue : page blanche ou connexion refusée</summary>

Vérifiez que `webServer.command` démarre bien le serveur sur le port 3000 et que
`public/index.html` est servi par `express.static("public")`. Avec
`reuseExistingServer: true`, un serveur déjà lancé est réutilisé.
</details>

<details>
<summary>Le stub sinon "fuit" entre les tests</summary>

Appelez `sinon.restore()` dans `afterEach`. Si vous stubbez des objets partagés,
préférez recréer des doublures locales (objets anonymes) dans chaque test.
</details>
